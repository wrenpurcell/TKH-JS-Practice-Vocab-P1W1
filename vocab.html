Vocab

  • The Internet

    The internet is a network of computers - clients and servers - all interconnected via fiber optic cables, wires, and wireless connections. Access to the internet is managed by internet service providers like Verizon or Optimum.

  • HTTP

    HTTP stands for HyperText Transfer Protocol. This was a set of rules or standards that defined how to access webpages via the internet. HTTPS later came as a more secure version of the protocol.

  • Client

    A client is a computer or program (like a browser) that makes a request for data like a webpage for example from another computer (usually a server). That data is then presented and or manipulated on the client computer.

  • Server

    A server is a computer that waits for requests for data from clients. The server then makes the request for the data from a database and then sends the data to the client.

  • Web Browser

    A web browser is a program/application that displays webpages to the user. It handles visually displaying html, css, and js files. A web browser typically has a search bar where a user can type a URL to make an HTTPS request for a specific website.

  • Website

    A website is a collection of webpages built to work together and are logically grouped. A typical website has a home page and can have an about page as well as many others.

  • Web page

    A web page is the rendering of typically one html file linked to css and js files. A web page can stand alone or be part of a website.

  • JavaScript

    JavaScript is a programming language originally designed to add functionality to webpages. With JS now buttons and forms can be interactive, rather than just plain html and css. With the development of Node.js JavaScript became even more powerful and can now be run outside of a browser making it just as powerful and useful as other programming languages.

  • HTML

    HTML stands for HyperText Markup Language. It is the language the defines how a webpage is to be structured. HTML uses tags to define HTML elements like head and body among others. HTML defines semantic meaning to webpage elements like separating articles or paragraphs and headings as opposed to just using a div tag for everything.

  • CSS

    CSS stands for Cascading Style Sheet. While HTML defines the structure of a webpage, CSS defines the layout and design of the webpage. With CSS you can change background colors, create a grid, and change how elements are displayed.

  • URL

    URL stands for Uniform Resource Locator. A URL is like an address for which to find a certain website. A URL is composed of a protocol, domain, followed by a path, query, and/or hash. This defines the which server the request for information goes to.

  • IDE / text editor

    IDE stands for Integrated Development Environment. As the name suggests, it is a program that allows developers to code usually with added functionality like code completion and underlining code that is syntactically incorrect. Most IDEs allow developers to run code and see output all in one program.

  • Command line / Terminal

    The terminal is a program the simulates a physical terminal where computer users would type commands. The command line is a text based interface with which to interact with the computer. On the command line you can easily list, create, and manipulate files and directories among other powerful functionality.

  • BASH

    BASH is a common language for interacting with the computer via the terminal. Other languages include ZSH. BASH stands for Bourne-Again Shell. It is a command language interpreter.

  • Operating System

    An operating system is a piece of software that allows the user of the computer to interact with the hardware. The operating system manages things like directories, RAM/memory, and (usually) provides a graphical user interface for the user.

  • Algorithm

    An Algorithm is any detailed list of steps for completing a task. This could be an algorithm/program that decides what videos Youtube recommends to a user or a recipe or list of steps it takes to cook something.

  • Frontend vs Backend Development

    Frontend Development deals with how a webpage or web app displays on a browser. This includes creating and updating html, css and js files and maybe working with frameworks like React. Backend Development works as the name would suggest behind the scenes, making requests to databases, creating databases, storing, accessing, and manipulating stored data.

    A simple way of thinking about it is front end is client side and backend is server side.

  • Programming vs Coding

    Coding is the language that helps translate human instructions into a way computers can understand. Programming is the use of code to create a program or set of instructions to accomplish some task.

  • Git vs Github

    Git is a version control system. It works by saving different versions of files as lines and various versions as additions or subtractions of those lines. This makes git very powerful for reverting back to previous versions, forking different branches and working on projects with multiple people and teams. Github on the other hand is just one of various online places to store repositories (or folders with code) that adds extra functionality to git.

  • Github terms: push, pull, fork, clone, repository

    Push

    To push is to upload your code and all files in a repository to the cloud.

    Pull

    To pull is to save changes from the cloud to your local repository.

    Fork

    To fork is to start a new branch off another branch. This means all new commits to the forked branch will not affect the original branch unless a pull request is made and committed.

    Clone

    To clone is to make a copy of a cloud based repository to your local system. You can then make changes and push to that cloud based repository.

    Repository

    A repository is a collection of files that are being tracked by a version control system like git.
